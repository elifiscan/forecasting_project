4. Custom Function for Required Accuracy Measures

To strictly comply with the project rubric, a custom evaluation function is defined to extract the required metrics: Bias (Mean Error), MAD, MSE, MAPE, RSFE, and Tracking Signal.

calculate_metrics <- function(actual, fitted_vals) {
  e <- actual - fitted_vals
  bias <- mean(e, na.rm = TRUE)
  mad_val <- mean(abs(e), na.rm = TRUE)
  mse_val <- mean(e^2, na.rm = TRUE)
  mape_val <- mean(abs(e / actual), na.rm = TRUE) * 100
  rsfe_val <- sum(e, na.rm = TRUE)
  tracking_signal <- rsfe_val / mad_val
  
  return(c(Bias = bias, MAD = mad_val, MSE = mse_val, MAPE = mape_val, RSFE = rsfe_val, Tracking_Signal = tracking_signal))
}

5.7 Holt-Winters (Multiplicative Decomposition equivalent)

To capture both trend and seasonality simultaneously, Holt-Winters is utilized. Multiplicative seasonality is chosen because seasonal variance expands with the index level. Additive decomposition is skipped in favor of this multiplicative approach.

hw_model <- hw(ciro_ts, seasonal = "multiplicative", h = 1)
metrics_hw <- calculate_metrics(ciro_ts, fitted(hw_model))

7. Superior Method Selection & Final Forecast

Based on the quantitative error metrics (specifically having the lowest MAD and MAPE) and its theoretical alignment with the underlying data structure (strong positive trend + expanding seasonality), the Holt-Winters Multiplicative Method is selected as the superior method.

Models lacking a trend parameter (Naive, SES) systematically under-forecasted the momentum, resulting in poor Tracking Signals.

final_prediction <- hw_model$mean[1]
